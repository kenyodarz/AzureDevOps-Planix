# Skill: traceability

## When to use it

Use this throughout the initiative, especially before closing work or publishing the portable repo.

## Objective

Ensure that every important requirement can be traced to:

- spec
- model or entity
- use case
- entry point or exposure primitive
- adapter
- critical configuration
- tests

## Main artifact

Use `docs/templates/traceability-matrix.md`.

## Core procedure

1. Assign IDs to functional and non-functional requirements.
2. Link each requirement to the spec artifact where it originates.
3. Relate it to model, use case, entry point, adapter, config, and tests.
4. Update the status as work progresses.

## Minimum closure rule

Do not consider an initiative complete if traceability is incomplete.
